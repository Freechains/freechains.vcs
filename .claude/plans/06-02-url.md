# remote URL defaults (clone + sync)

## Purpose

Make every command that takes a remote ergonomic: a peer should be
reachable by **just an IP**.
Normalize a bare remote into a full `git://` URL using freechains
defaults, while leaving explicit URLs and local paths untouched.

```
freechains chains add '#chat' clone 1.2.3.4
        -> git clone git://1.2.3.4:8330/#chat
freechains chain '#chat' sync recv 1.2.3.4
        -> git fetch git://1.2.3.4:8330/#chat ...
freechains chain '#chat' sync send 1.2.3.4
        -> git push  git://1.2.3.4:8330/#chat ...
```

`git://` is the default because the README serves with `freechains
daemon` (a git daemon): its `--base-path=<root>/chains/` makes the URL
path just the alias, so the alias default composes cleanly.
`ssh://` would need a filesystem path on the remote and is therefore
left to the explicit form.

## Shared helper

One parser in `common.lua`, reused by all three call sites.

```lua
function url (raw, alias)
    if raw:match("^[/~.]") or raw:find("://") then
        return raw
    end
    local hostport, path = raw:match("^([^/]+)(/?.*)$")
    if not hostport:find(":") then
        hostport = hostport .. ":8330"
    end
    if path == "" then
        path = "/" .. alias
    end
    return "git://" .. hostport .. path
end
```

## Rules (applied in order)

| # | condition                              | action                         |
|---|----------------------------------------|--------------------------------|
| 1 | starts with `/`, `~`, `./`, `../`      | local path -> verbatim         |
| 2 | contains `://`                         | explicit scheme -> verbatim    |
| 3 | otherwise (bare `host[:port][/path]`)  | normalize (below)              |

Normalization for rule 3:

- scheme : prepend `git://`
- port   : if no `:port`, insert `:8330` (the `freechains daemon` default)
- path   : if no `/path`, append `/<alias>`

## Examples (alias `#chat`)

| input                  | normalized                       |
|------------------------|----------------------------------|
| `1.2.3.4`              | `git://1.2.3.4:8330/#chat`       |
| `1.2.3.4:9999`         | `git://1.2.3.4:9999/#chat`       |
| `1.2.3.4/#other`       | `git://1.2.3.4:8330/#other`      |
| `host/#chat`           | `git://host:8330/#chat`          |
| `git://h:8330/#chat`   | verbatim (explicit scheme)       |
| `ssh://bob@h/p`        | verbatim (explicit scheme)       |
| `/srv/chains/#chat`    | verbatim (local path)            |
| `./peer/chains/#chat`  | verbatim (local path)            |

## Call sites

| File                            | Place        | Change                                  |
|---------------------------------|--------------|-----------------------------------------|
| `src/freechains/common.lua`     | new `url()`  | the normalizer above                    |
| `src/freechains/chains.lua`     | `clone`      | `git clone (url(ARGS.url, ARGS.alias))` |
| `src/freechains/chain/sync.lua` | `ARGS.send`  | push to `url(ARGS.remote, ARGS.alias)`  |
| `src/freechains/chain/sync.lua` | `ARGS.recv`  | fetch from `url(ARGS.remote, ARGS.alias)` |

`ARGS.alias` is present for both `chains add` and `chain <alias> sync`.

## Notes

- The remote path defaults to the **same alias** being used locally; it
  resolves on the remote because `freechains daemon` serves both the
  `<alias>` symlink and the `<hash>` dir under `--base-path`.
  If the remote uses a different alias, pass `/<their-alias>` or the
  `<hash>` explicitly.
- `#` in the path is fine over `git://` (verified).
- The friendly default is `git://` (anonymous). For `send`, the remote
  must be a writable hub (`freechains daemon --hub`).
  Authenticated/private peers pass an explicit `ssh://...` (rule 2).

## Open questions

- `user@host` bare input implies SSH but rule 3 would build a malformed
  `git://user@host:...`.
  Detect `@` and require explicit `ssh://`, or reject?
- Default port: hard-code `8330`, or read a config/`--port`?
  `8330` mirrors the `daemon` default ([[06-01.daemon]]).
- Relative local paths without a leading `./` (e.g. `peer/x`) are read as
  a host; document that local paths need `/`, `~`, or `./`.
