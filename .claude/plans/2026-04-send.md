# Plan: freechains chain sync send

## Context

- recv done; send is stub
  (`src/freechains/chain/sync.lua:5` — `error "TODO"`)
- choice: push + remote `post-receive` hook runs recv
- no begs yet; send fails non-zero on rejection
- tests: `tst/cli-send.lua` (currently a recv copy — convert)

## Problem: working-tree push

Chain repos are non-bare (posts live as files).
`git push` to the checked-out branch is refused by default.

## Approach: push to side ref

1. Sender:
    - `git push <remote> main:refs/incoming/<tag>`
2. Remote `post-receive` hook:
    - run recv-equivalent with `ref=refs/incoming/<tag>`
      (no `git fetch`; data already local)
    - consensus + validation + replay
    - update `refs/heads/main` on success
    - delete `refs/incoming/<tag>`
    - exit 0 accept / non-zero reject
3. Sender reads hook stderr; non-zero -> send fails

### 5. Test: send conflict (loser commit voided)
