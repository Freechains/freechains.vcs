# Plan: freechains chain sync send

## Context

- recv done; send is stub
  (`src/freechains/chain/sync.lua:5` — `error "TODO"`)
- choice: push + remote `post-receive` hook runs recv
- no begs yet; send fails non-zero on rejection
- tests: `tst/cli-send.lua` (currently a recv copy — convert)

## Problem: working-tree push

Chain repos are non-bare (posts live as files).
`git push` to the checked-out branch is refused by default.

## Approach: push to side ref

1. Sender:
    - `git push <remote> main:refs/incoming/<tag>`
2. Remote `post-receive` hook:
    - run recv-equivalent with `ref=refs/incoming/<tag>`
      (no `git fetch`; data already local)
    - consensus + validation + replay
    - update `refs/heads/main` on success
    - delete `refs/incoming/<tag>`
    - exit 0 accept / non-zero reject
3. Sender reads hook stderr; non-zero -> send fails

#### Hook location

`<repo>/.git/hooks/post-receive`
(outside working tree — `skel/` won't reach it)

#### Hook source

New file `src/freechains/hooks/post-receive`
(sibling of `skel/`). Shipped shell script, exec bit set.
`chains.lua` copies it and `chmod +x`.

#### Hook body (sketch)

```sh
#!/bin/sh
# stdin: <old> <new> <ref> per updated ref
while read old new ref; do
    case "$ref" in
    refs/incoming/*)
        freechains --root="$FC_ROOT" \
            chain "$FC_CHAIN" sync _accept "$new" "$ref" \
            || exit 1
        ;;
    refs/heads/*)
        echo "ERROR : chain sync : push to $ref not allowed" 1>&2
        exit 1
        ;;
    esac
done
```

#### Hook environment

Derive from `$GIT_DIR`:

- `$GIT_DIR = <root>/chains/<hash>/.git`
- `FC_ROOT  = $GIT_DIR/../../..`
- `FC_CHAIN = $(basename $(dirname $GIT_DIR))`

#### Side-ref concern

Push to `refs/incoming/<tag>` avoids
`receive.denyCurrentBranch`.
After accept, hook runs `git update-ref refs/heads/main`
plus `git checkout-index -fa` (or `git reset --hard`) to
refresh the working tree.
