  (`src/freechains/chain/sync.lua:5` — `error "TODO"`)

- no begs yet; send fails non-zero on rejection
- assumption: working tree is always clean (no uncommitted posts)
- tests: `tst/cli-send.lua` (currently a recv copy — convert)

## Approach: push main, updateInstead

1. `chains.lua` sets
   `git config receive.denyCurrentBranch updateInstead`
   (accept push to checked-out `main`; git refreshes WT).
2. Sender: `git push <remote> main`.
3. Remote `pre-receive` hook:
    - run recv-equivalent with new tip
      (no `git fetch`; data already local)
    - consensus + validation + replay
    - exit 0 accept / non-zero reject
4. Sender reads hook stderr; non-zero -> send fails
   (push aborts, ref unchanged).

`chains.lua` inlines a `cp` after `git_config(tmp)`.

#### Hook body (final sketch — grows step by step)

```sh
#!/bin/sh
while read old new ref; do
    case "$ref" in
    refs/heads/main)
        freechains --root="$FC_ROOT" \
            chain "$FC_CHAIN" sync _accept "$new" \
            || exit 1
        ;;
    *)
        echo "ERROR : chain sync : only main push allowed" 1>&2
        exit 1
        ;;
    esac
done
```

#### Hook environment

Derive from `$GIT_DIR` (set by git when hook runs):

- `$GIT_DIR = <root>/chains/<hash>/.git`
- `FC_ROOT  = $GIT_DIR/../../..`
- `FC_CHAIN = $(basename $(dirname $GIT_DIR))`

#### Config: updateInstead

`git_config(dir)` also runs:

```
git -C <dir> config receive.denyCurrentBranch updateInstead
```

Accepts push to checked-out `main` and refreshes WT
(requires clean WT — our invariant).
