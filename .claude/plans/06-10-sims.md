# Replicate TPD-21 Performance Experiments

## Goal

Replicate the four §3.4 experiments from `tpd-21` against the
current Lua/Git rewrite (`freechains-dev-2`).

Source experiments live in `/x/papers/tpd-21/{chat,usenet}/`.

## Status: Planning

## Verdict

Replicable in principle.
Methodology ports cleanly; absolute numbers will NOT match
because the reps rules and CLI changed since the paper.
Replication validates feasibility, not the paper figures.

## What the original sims do

Single-host replay of public archives, one block per message:

1. create an ed25519/GPG key the first time a user appears
2. set daemon clock to the message archive timestamp
3. post signed by that user
4. if `BLOCKED`, the fixed pioneer likes it to unblock

Forks (item c) come from a separate multi-host driver
(`chat-03.sh` all-pairs `send;recv`), not the `*-02.lua` scripts.

| file                  | role                                  |
|-----------------------|---------------------------------------|
| `chat/chat-02.lua`    | Wikimedia IRC replay (single host)    |
| `usenet/use-02.lua`   | comp.compilers mbox replay            |
| `common.lua`          | harness: `post()` + auto-like         |
| `chat/chat-03.{lua,sh}` | multi-host fork driver              |

## Old CLI -> new CLI mapping

| Sim call (old Kotlin)         | New Lua/Git                          | Risk |
|-------------------------------|--------------------------------------|------|
| `freechains-host --port=N start` | `freechains --root=/tmp/N daemon` | low  |
| `now <ms>` (daemon clock)     | `--now=<sec>` per command            | low  |
| `keys pubpvt 'pass'`          | `ssh-keygen -t ed25519 -f /tmp/u`    | low  |
| `chains join '$chat' SHA`     | `chains add '#chat' init` / `clone`  | low  |
| `post inline/file P --sign`   | `post inline/file P --sign=/tmp/u`   | low  |
| `chain X like HASH` (unblock) | post-like path unclear               | high |
| `chain X heads blocked`       | `begs` mechanism                     | med  |
| `chain X consensus`           | `chain X list order`                 | low  |
| `peer localhost:N send/recv`  | `chain X sync send/recv <remote>`    | low  |

## Risks / caveats

- Reps rules changed (transfer tax: 10 sent -> 9 recv, sender
  29 -> 20).
  Paper constants (30 reps, 24h award, 12h penalty) likely
  differ -> quantitative results diverge.
- Welcoming-like on a blocked post is the main CLI unknown.
- Harness must be rewritten for ssh keys, `--root`, `--now=sec`.
- Archives still fetchable from archive.org:
  `WikimediaIrcLogs` and `usenet-comp` (URLs in `use-02.lua`).

## Four evaluation items

| item | metric            | paper chat | paper news |
|------|-------------------|------------|------------|
| a    | metadata overhead | 10x        | ~41% (50%) |
| b    | consensus init/incr | 125s/50ms | ~65s/70ms  |
| c    | fork ratio        | 18%        | 14%        |
| d    | blocked ratio     | 3.7%       | 3.5%       |

Paper vs raw-log discrepancies to keep in mind: news (b) logged
~65s not 100s; news (d) was 8.9% pre-fix, 3.5% post-fix.
