## Verified mechanism

`sync send` does NOT push content. Per the hook
`lua/freechains/hooks/pre-receive:42-47`, a push TRIGGERS the hub to
run `sync recv` back from the sender's `url`; the hub does the full
recv (consensus + merge) locally, then rejects the push with
`Freechains: OK` (treated as success by `sync.lua:13`).

Consensus winner: `sync.lua:78-112` — the branch whose signers hold
more summed reputation at the common ancestor wins; tie -> smaller
commit hash. So when B sends a divergent branch to hub X, X resolves
consensus on itself.

Do NOT document (unimplemented / contradicts code): 7-day / 100-post
hard fork; `consensus.md`'s "earlier author date wins".

## Caveats to handle when drafting / running

1. Same-machine only: pushed `url` is the sender's local repo path, so
   the hub recvs from a filesystem path. Fine for this single-machine
   guide; note it is not a cross-network demo.
2. Read the result locally on X (`--root=/tmp/X/ list order`) — a
   `--hub` daemon serves receive-pack only, so you cannot pull from X.
3. Three terminals: A daemon, X `--hub` daemon, command shell.
4. Clean divergence: at this point B already carries the Charlie-like
   commit, so B's branch is `[charlie-like, bob-post]` vs A's
