# `freechains chain <alias> all` command

Group `order` and a new `dag` view under a single `all` subcommand.
The `all` namespace covers reads that traverse the full chain history.

## CLI shape

```
freechains chain <alias> all order
freechains chain <alias> all dag
```

- `all order` : same output as the current `chain <alias> order`
- `all dag`   : ASCII DAG of non-state commits, 40-col wide,
                top-to-bottom, genesis centered at top.
                Strict alternation of hash row and lane row.
                Siblings (same non-state parent) share one
                hash row, side-by-side. Lane offset K=4
                (centers at parent_col ± 4, hashes have a
                1-col gap). Connectors:
                  - linear: `|`
                  - fork  : `/   \` (3 spaces between)
                  - join  : `\   /` (currently unreachable —
                    no non-state merges in this architecture)
                Rows are right-trimmed of trailing whitespace.

## `chain/all.lua` (shape)

```lua
if ARGS.order then
    -- print non-state hashes from G.order
elseif ARGS.dag then
    -- render 40-col DAG (see algorithm below)
end
```

Decision: `order` and `dag` are inlined directly in `all.lua`
(no separate `order.lua` / `dag.lua` files).

## DAG render algorithm

### Inputs

- `G.order`     : consensus-ordered hashes (oldest → newest)
- `trailer(h)`  : returns `"state"` for state commits
- git parents   : `git -C REPO log --format='%H %P' <hash>`

### Steps

1. Build the filtered list
    - keep only `G.order` entries where `trailer(h) ~= "state"`
    - call this list `V` (top-to-bottom render order)

2. Compute non-state parents per node
    - for each `h` in `V`, get raw git parents
    - for each raw parent, walk back through state commits until a
      non-state ancestor is found
    - store `parents[h]` as the list of non-state ancestors (1 or 2)

3. Column assignment (single pass over `V`)
    - constants: `WIDTH = 40`, `MID = 20`
    - `col[V[1]] = MID` (genesis at center)
    - for each subsequent `h`:
        - 1 parent
            - if parent already has another assigned child, fork:
                place `h` at parent col offset by next free `±step`
            - else inherit parent col
        - 2 parents (merge)
            - place `h` at midpoint of parent cols
            - if `h` is the last node (tip), force `MID`
    - clamp every assignment to `[0, WIDTH-1]`

4. Render
    - one row per commit, in `V` order
    - put `*` at `col[h]`; pad with spaces
    - between consecutive commits insert one connector row
      drawing `|`, `/`, `\` from each parent col to child col
      (parent above, child below: `\` if child col > parent col,
       `/` if child col < parent col, `|` if equal)
    - for a merge, draw two connectors converging on the merge col

5. Print rows top-to-bottom, each line trimmed to `WIDTH`

### Output shape (40-col strip, `.` shown as space)

```
0........10........20........30........39
                    *                      <- first
                    |
                   * *                     <- fork
                   | |
                   | *
                    \|
                    *                      <- merge
                    |
                    *                      <- tip
```

## Out of scope

- Color, terminal width detection, paging
- Showing state commits (filtered out by design)
- Edge labels, hash prefixes next to nodes
- Branch names or remote refs

## Open questions

- Fork direction: deterministic left-then-right vs. nearest-free
- Merge of two distant columns: render zig-zag connectors over
  multiple intermediate rows, or single diagonal collapsing in
  one row? (current sketch: single row)

## Cross-cutting issue: `get metadata` over a sync merge

### Symptom

`freechains chain X get metadata <LIKE>` crashes when `<LIKE>`
is a like-on-beg whose ancestry passes through a state merge `M`
from `sync.lua` case-4 diverge. Minimal repro: `tst/cli-get-merge.lua`.

### Root cause

Two independent issues compound:

1. `sync.lua:392` creates `'x'` merge commits with `git commit -m 'x'`
   — no `Freechains:` trailer. `get.lua`'s `rec` walker hits these
   and falls into the `else error("bug found : invalid trailer")`.

2. Even if `'x'` is fixed, `rec` walks transitively through every
   `state` ancestor, accumulating post/like ancestors from BOTH
   sides of any prior fork. For a like-on-beg over a sync merge,
   `#backs` reaches 3+, blowing `assert(#backs <= 2)`.

### Three reconciliation paths

| Choice | Effect on `backs` | Effect on dag |
|---|---|---|
| **A.** Keep recursion; drop the `<= 2` assert | unbounded; `backs` can include 3+ post/likes from prior forks | unchanged |
| **B.** Stop recursion at `state`-with-≥2-parents (treat `M` as a back) | `<= 2` holds; `backs` may contain state-merge hashes that aren't queryable via `get metadata` | unchanged |
| **C.** Eliminate `'x'`/`M` in sync (cherry-pick loser instead of merging) | clean app DAG; `backs` ≤ 2 naturally | dag loses `*` nodes; only post/like |

### Decision: option A (drop assert; `backs` walks freely)

Implemented in 3 steps:
- Step 1: `backs()` factored into `common.lua`; `get.lua` uses it; assert removed; latent `nil`-author bug in `apply` discount-scan guarded.
- Step 2: `'x'` sync intermediates carry `Freechains: merge` trailer; `backs()` recurses through them like `state`.
- Step 3: `chain/all.lua` dag uses `backs()`; V = post/like only; no more `*` glyph; state-merges invisible.

### If decision lands on A or B

Refactor for a single source of truth:

| File | Change |
|---|---|
| `src/freechains/chain/walk.lua` (new) | exports `backs(hash)` — the canonical "logical parents" of a commit |
| `src/freechains/chain/get.lua` | replaces inline `rec` with `walk.backs(...)` |
| `src/freechains/chain/all.lua` (dag block) | replaces `vparents`/`up_v` with `walk.backs(...)`; same traversal as `get` |

Also fix `'x'` regardless: add `--trailer 'Freechains: state'` at
`sync.lua:392` so the walker never sees a nil-trailer commit.

### If decision lands on C

Bigger sync refactor; defer DAG work until sync is settled.
