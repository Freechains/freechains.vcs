# freechains daemon

## Purpose

Serve local chains to peers over the network without exposing raw
`git daemon` flags.
Wraps `git daemon` with freechains-correct defaults (base-path,
export-all, port), and gates writability behind an explicit `--hub`.

## CLI

```
freechains daemon [--port=<n>] [--hub] [-- <raw git daemon args>]
```

| option       | effect                                              |
|--------------|-----------------------------------------------------|
| (default)    | read-only (`upload-pack`): serves clone + `sync recv` |
| `--hub`      | also enable `receive-pack`: accepts `sync send` pushes |
| `--port=<n>` | listen port (default 8330)                          |
| `-- <args>`  | escape hatch: forwarded verbatim to `git daemon`    |

Foreground process; Ctrl-C to stop.
Everything after `--` is collected (`ARGS.xtra`) and appended to the
`git daemon` command, e.g. `--detach --pid-file=<f> --listen=127.0.0.1
--reuseaddr`.
The `--` is required so argparse does not reject the raw `--flags`.

## Roles

| mode      | git daemon flags                     | enables            |
|-----------|--------------------------------------|--------------------|
| default   | `--export-all --enable=upload-pack`  | clone, `sync recv` |
| `--hub`   | `--export-all --enable=receive-pack` | + `sync send`      |

`--enable=upload-pack` is the neutral filler: `upload-pack` is on by
default, so enabling it explicitly is a no-op. This keeps the flag shape
uniform (always one `--enable=`), with `receive-pack` the only behavior
change.

A **hub** is a writable rendezvous peer: many peers push their content
to it; it aggregates and re-serves to subscribers.
A plain daemon is a read-only source that subscribers pull from.

## Mapping

```
git daemon \
    --base-path=<root>/chains/ \
    --export-all \
    --enable=<upload-pack|receive-pack> \   # receive-pack only with --hub
    --port=<port>
```

`<root>` = `ARGS.root` (default `~/.freechains/`).

## Why `--hub` gates receive-pack

Anonymous push is safe **only because** the receiver's `pre-receive`
hook validates every push via `sync recv`: signatures, reputation,
consensus, and FF/divergence rules.
An invalid or malicious push is rejected; it cannot forge reputation or
rewrite history.

This holds **only if the hook actually runs**.
The hook was being installed non-executable and silently ignored
(fixed: `chains.lua` now `chmod +x` the copied hook on `init`/`clone`).
The invariant is therefore established at install time, not re-checked at
daemon start: a missing exec bit at that point would mean the repo was
corrupted after creation — the same category as the hook file or the
source itself being mangled, which we do not guard against. No startup
scan.

Read-only is the default so the writable footgun requires intent.
For authenticated writes without anonymous exposure, peers may push over
SSH instead (sshd already runs the hook; no daemon needed).

No separate module: the command is small enough to live inline in the
dispatch chain alongside `chains`/`chain`.

## dispatch (shape)

```lua
elseif ARGS.daemon then
    os.execute (
        "git daemon --base-path=" .. (ARGS.root .. "/chains/") ..
            " --export-all" ..
            " --enable=" .. (ARGS.hub and "receive-pack" or "upload-pack") ..
            " --port=" .. (ARGS.port or 8330) ..
            " " .. table.concat(ARGS.xtra, " ")
    )
end
```

`cmd.daemon._:argument("xtra"):args("*")` captures the post-`--` tokens.

Use `os.execute` (not `exec`): the daemon runs in the foreground and
must pass stdio through to the terminal, not be captured by `io.popen`.
